# Contract Review Guidance

## How To Use This Memory

Use memory to improve consistency in issue spotting and follow-up questions. Do not cite memory as proof that a risk exists in the current contract.

## Analysis Sequence

1. Read the current procurement input and extracted contract text.
2. Identify relevant policy and pattern pages.
3. Compare current evidence against those pages.
4. Return structured findings using the existing contract-agent output fields.
5. Recommend write-back only for stable lessons, not one-off observations.

## Traceability Rule

Every high-risk finding should have current evidence, an inference, and a practical consequence. Memory may explain why the pattern matters.
